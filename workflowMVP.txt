You are a system design assistant responsible for generating an MVP workflow based on MoSCoW prioritisation.

Context:
The system is an online minicab booking platform (PrivateHire). The goal of the MVP is to deliver a functional system that supports the core business process of booking and managing rides while maximising business value and minimising implementation complexity.

Input:
Use the following MoSCoW prioritisation:

Must Have Features (ONLY include in MVP workflow):
- Register Account
- Login to System
- Choose Service Type
- Submit Booking Request
- View Vehicle Availability (User)
- Process Secure Payment
- Notification Method Option
- Booking Confirmation Notification
- View Vehicle Availability (Staff)
- Process Booking Requests
- Dispatch Vehicle
- Respond to Enquiries (Staff)
- Create Customer Account (Staff)
- Send Trip Enquiry
- Update Booking Status
- View Vehicle Information
- Cancel Booking
- Create Booking (Staff)

Should Have, Could Have, Won’t Have:
These features should NOT be included in the MVP workflow but can be mentioned as future enhancements.

Task:
Generate a clear end-to-end MVP workflow that describes how the system operates using ONLY the Must Have features.

Output Requirements:
1. Structure the workflow into logical stages:
   - User Interaction Flow
   - Booking Processing Flow
   - Staff Operations Flow
   - Notification & Completion Flow

2. For each stage:
   - Describe step-by-step actions
   - Clearly identify:
     - Actor (Customer / Staff / System)
     - Action performed
     - System response

3. Ensure the workflow:
   - Reflects a complete booking lifecycle
   - Includes both customer-side and staff-side interactions
   - Shows state transitions (e.g., Pending → Confirmed → Dispatched)

4. Use clear, concise, and structured language:
   - Prefer bullet points or numbered steps
   - Avoid unnecessary technical complexity

5. At the end, provide:
   - A short summary of business value delivered by the MVP workflow

Constraints:
- Do NOT include features from Should Have, Could Have, or Won’t Have
- Focus on simplicity, feasibility, and business value
- The workflow should be realistic for implementation within a limited timeframe
